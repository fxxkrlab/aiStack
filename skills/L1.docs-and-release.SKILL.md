# L1 Skill: Docs and Release

## When to use

Use for architecture docs, runbooks, changelogs, and release notes.

## Workflow

1. Capture behavior change and operational impact.
2. Update docs nearest to changed code.
3. Keep examples runnable.
4. Add concise migration notes when required.

## Quality Gates

- Reader can execute steps without hidden assumptions.
- Version-sensitive statements are dated or sourced.
- All commands are copy-paste safe.
